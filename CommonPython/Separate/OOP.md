# ООП в Python — краткий конспект

## Что такое ООП?

ООП (объектно-ориентированное программирование) — подход, при котором программа строится вокруг объектов, объединяющих данные (атрибуты) и поведение (методы).

---

## Принципы ООП

### 1. Инкапсуляция
Сокрытие внутренней реализации объекта за публичным интерфейсом.

### 2. Наследование
Создание новых классов на основе существующих.

### 3. Полиморфизм
Единый интерфейс для объектов разных типов.

### 4. Абстракция
Сокрытие сложности реализации за простым интерфейсом.

---

## Доступ к атрибутам и методам

При обращении к `obj.attr` Python ищет атрибут в порядке:

1. Дескрипторы данных
2. `obj.__dict__`
3. `Class.__dict__`
4. Базовые классы (MRO)
5. `__getattr__`

Для методов:

```python
obj.method()
```

эквивалентно:

```python
Class.method(obj)
```

## Что такое self?

`self` — ссылка на текущий экземпляр класса.

```python
class User:
    def __init__(self, name):
        self.name = name
```

---

## Магические методы

Специальные методы вида:

```python
__init__
__str__
__repr__
__len__
__iter__
__add__
```

Позволяют объектам участвовать во встроенных операциях Python.

---

## Что такое super()?

Позволяет обращаться к следующему классу в цепочке MRO.

```python
super().__init__()
```

Особенно важно при множественном наследовании.

[MRO](/MRO.md)

---

## Что такое дескрипторы?

Объекты, реализующие хотя бы один метод:

```От себя
Просто обычные getter и setter из c#
```

```python
__get__
__set__
__delete__
```

На дескрипторах построены:

- property
- classmethod
- staticmethod
- ORM-поля

---

## Что такое абстрактные классы?

Классы, которые нельзя инстанцировать напрямую.

Используются через модуль `abc`:

```python
from abc import ABC, abstractmethod
```

Позволяют задавать обязательный интерфейс для потомков.

---

## Когда использовать __slots__?

Используется для экономии памяти и ограничения набора атрибутов.

```python
class User:
    __slots__ = ("name", "age")
```

Подходит, если создаются миллионы однотипных объектов.

[SLOTS](/Slots.md)

---

## Когда использовать метаклассы?

Метаклассы управляют созданием классов.

Основные применения:

- регистрация классов
- ORM
- автоматическая валидация
- генерация атрибутов

В большинстве случаев достаточно декораторов или `__init_subclass__`.

## Разница между __dict__ класса и экземпляра

### Class.__dict__

Содержит пространство имён класса:

- атрибуты класса
- методы
- служебные объекты

### instance.__dict__

Содержит данные конкретного экземпляра:

```python
u.__dict__
# {'name': 'Alex'}
```

### Важный пример

```python
class User:
    role = "admin"

u = User()
```

```python
User.__dict__["role"]  # есть
u.__dict__             # role нет
u.role                 # доступен через класс
```

Экземпляр видит атрибуты класса через механизм поиска атрибутов, но они не копируются в его `__dict__`.
