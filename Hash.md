# Что такое хэш-таблица?
Хэш-таблица — это структура данных, которая позволяет очень быстро (в среднем за O(1)) находить, добавлять и удалять элементы.
- Главная идея: использовать хэш-функцию, чтобы вычислять, где именно должен лежать элемент в памяти.

Как это работает (простыми словами):

Представь большой массив (список) с ячейками:

Индекс:   0    1    2    3    4    5    6    7    ...

Ячейки:  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]  ...

Вычисляется хэш от ключа: hash(key) → например, число 123456.
Это число преобразуется в индекс массива (например, 123456 % размер_массива).
Значение сохраняется в эту ячейку.

# Что такое Dict

Словарь (dict) в Python — это реализация хэш-таблицы (hash table).
Как работает хэш-таблица:

1. Когда ты пишешь d[key] = value:
Python вызывает hash(key) — получает хэш-код (целое число).
Этот хэш-код преобразуется в индекс в массиве (таблице).
По этому индексу сохраняется пара (key, value).

2. При поиске d[key]:
Снова вычисляется hash(key).
Python быстро переходит по индексу и проверяет ключ с помощью __eq__.

### Преимущества хэш-таблицы:

В среднем O(1) (очень быстро) для операций добавления, поиска и удаления.
Не зависит от количества элементов (почти).

Недостатки:

Иногда бывают коллизии (одинаковый хэш у разных ключей) — Python их решает методом цепочек (открытая адресация + списки).
Памяти занимает больше, чем список.

# ⭐ Что может быть ключами словаря?
Ключом в словаре может быть только хэшируемый объект, то есть такой, у которого:

Реализован метод __hash__() (возвращает int)
Реализован метод __eq__()
Хэш объекта не меняется за время его жизни

# Что такое множество (set)?
Множество — это неупорядоченная коллекция уникальных элементов.
Внутри тоже используется хэш-таблица, но хранятся только сами элементы (без значений).
Основные особенности set:

Хранит только уникальные элементы (дубликаты автоматически удаляются).
Очень быстрый поиск (in), добавление и удаление — O(1) в среднем.
Элементы должны быть хэшируемыми.
Неупорядоченное (до Python 3.6), с Python 3.7+ сохраняет порядок вставки.

```Python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        """Определяем, когда две точки считаются равными"""
        if isinstance(other, Point):
            return self.x == other.x and self.y == other.y
        return False

    def __hash__(self):
        """Определяем хэш на основе координат"""
        return hash((self.x, self.y))   # кортеж из неизменяемых значений


# ==================== Пример использования ====================

p1 = Point(3, 4)
p2 = Point(3, 4)
p3 = Point(5, 6)

print(p1 == p2)           # True
print(hash(p1) == hash(p2))  # True
print(p1 == p3)           # False

# Можно использовать в set и dict
points = {p1, p2, p3}
print(len(points))        # 2 (p1 и p2 считаются одним и тем же)

d = {}
d[p1] = "точка A"
d[p2] = "точка B"         # перезапишет значение, т.к. p1 == p2
print(d[p1])              # "точка B"
```
