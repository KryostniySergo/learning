# `__slots__` в Python

## Что такое `__slots__`?

`__slots__` — это специальный механизм Python, который позволяет отказаться от создания `__dict__` у экземпляров класса и хранить атрибуты в заранее выделенных слотах памяти.

Основная цель:

- уменьшить потребление памяти;
- немного ускорить доступ к атрибутам;
- запретить создание произвольных новых атрибутов.

---

## Как атрибуты хранятся по умолчанию?

```python
class User:
    pass
```

```python
u = User()

u.name = "Alex"
u.age = 25
```

Все атрибуты сохраняются в словаре объекта:

```python
print(u.__dict__)
```

Результат:

```python
{
    "name": "Alex",
    "age": 25
}
```

Схематично:

```text
User instance
    |
    └── __dict__
            |
            ├── name -> Alex
            └── age  -> 25
```

Такой подход очень гибкий, но требует дополнительной памяти.

---

## Что делает `__slots__`?

```python
class User:
    __slots__ = ("name", "age")
```

Теперь объект может содержать только указанные атрибуты.

```python
u = User()

u.name = "Alex"
u.age = 25
```

Работает корректно.

Попытка добавить новый атрибут:

```python
u.city = "Riga"
```

Приведёт к ошибке:

```python
AttributeError
```

---

## Что изменилось внутри?

Теперь у экземпляра нет словаря атрибутов:

```python
u.__dict__
```

Ошибка:

```python
AttributeError
```

Вместо словаря используются фиксированные слоты памяти:

```text
User instance

slot #0 -> name
slot #1 -> age
```

---

## Почему это экономит память?

### Без `__slots__`

```text
instance
    |
    └── dict
            |
            ├── key
            ├── hash
            └── value
```

Каждый объект содержит собственный словарь.

### С `__slots__`

```text
instance
    |
    ├── slot 0
    └── slot 1
```

Словарь отсутствует, поэтому накладные расходы памяти меньше.

Особенно заметно при создании миллионов объектов.

---

## Пример

Без `__slots__`:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
```

С `__slots__`:

```python
class Point:
    __slots__ = ("x", "y")

    def __init__(self, x, y):
        self.x = x
        self.y = y
```

---

## Как реализован `__slots__` внутри Python?

При создании класса:

```python
__slots__ = ("x", "y")
```

Python создаёт специальные дескрипторы для каждого поля.

Можно увидеть:

```python
Point.__dict__["x"]
```

Результат будет примерно таким:

```python
<member 'x' of 'Point' objects>
```

Это специальный C-level дескриптор (`member_descriptor`).

Поэтому доступ:

```python
p.x
```

проходит через дескриптор, а не через `__dict__`.

---

## Можно ли вернуть `__dict__`?

Да.

```python
class User:
    __slots__ = ("name", "__dict__")
```

Теперь снова можно создавать произвольные атрибуты:

```python
u = User()
u.city = "Riga"
```

---

## Наследование и `__slots__`

### Родительский класс

```python
class A:
    __slots__ = ("x",)
```

### Потомок

```python
class B(A):
    pass
```

У класса `B` автоматически появится собственный `__dict__`.

Поэтому:

```python
b = B()
b.y = 10
```

работает.

---

Чтобы сохранить ограничения:

```python
class B(A):
    __slots__ = ()
```

или

```python
class B(A):
    __slots__ = ("y",)
```

---

## Когда использовать `__slots__`?

Подходит для объектов:

- создающихся миллионами экземпляров;
- имеющих фиксированный набор полей;
- не требующих динамического добавления атрибутов.

Типичные примеры:

```python
Point
Vector
Token
Node
ASTNode
GraphVertex
```

---

## Когда не использовать?

Обычно не стоит использовать:

- в бизнес-логике без проблем с памятью;
- в Django-моделях;
- в Pydantic-моделях;
- если требуется динамически добавлять атрибуты.

Во многих проектах выигрыш оказывается незначительным по сравнению с усложнением кода.

---

## Частый вопрос на собеседовании

### Что происходит при объявлении `__slots__`?

Правильный ответ:

> Python не создаёт `__dict__` для экземпляров класса, а вместо него генерирует набор специальных дескрипторов (`member_descriptor`), которые обращаются к заранее выделенным слотам памяти.

Это позволяет:

- уменьшить потребление памяти;
- немного ускорить доступ к атрибутам;
- ограничить набор допустимых атрибутов.

---

## Кратко для собеседования

`__slots__` позволяет хранить атрибуты не в `__dict__`, а в фиксированных слотах памяти.

Преимущества:

- меньше памяти;
- немного быстрее доступ к атрибутам;
- защита от случайного создания новых полей.

Недостатки:

- меньше гибкости;
- усложнение наследования;
- не подходит для объектов с динамической структурой.
